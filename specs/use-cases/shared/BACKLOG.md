# Backlog – Shared Use Cases

## Objetivo

Registrar fluxos e políticas transversais compartilhados entre múltiplos serviços.

## Itens de backlog (próxima fase)

- `UC-SHARED-001`: Correlação e auditoria ponta a ponta (agent -> MCP -> ACL).
- `UC-SHARED-002`: Política comum de idempotência para reexecução segura.
- `UC-SHARED-003`: Padronização de erro entre serviços internos.
- `UC-SHARED-004`: Estratégia de retry e compensação para falhas parciais.
